패킷을 보낼 때 규칙을 정리

첫 1바이트는 메뉴, 인벤, 싱글모드, 멀티모드 등, 모드를 식별 하기 위한 패킷
0x00 >> login >> 게임을 처음 틀었을 때, 로그인 인증 서버에 연결 하기 위한 패킷
0x01 >> lobby >> 로그인 인증에 성공하면 로비로 보낸다. 패킷 전송. 클라에 보내면 메뉴창이 뜸
0x02 >> singlePlay >> 싱글 플레이 버튼을 눌렀을 때 전송 (받음).
0x03 >> multiLobby >> 멀티 플레이 버튼을 눌렀을 때 전송 (받음). 처음엔 로비로 보내서 다른 매칭자가 나올때 까지 대기한다.
0x04 >> multiBattle >> 멀티 플레이가 성립 되었을 때 (전송). 전투 화면으로 이동한다.
0x05 >> multiCancle >>
0x06 >> battleConfirm >>
    내부 배틀 로직 함수 안에서 >>
    0x51 >> battleContinue >> 배틀을 계속 진행
    0x52 >> battleGiveup >> 접속을 종료했거나 기권을 했을 경우
    0x53 >> deckListReq >> 처음 접속 시 상대방의 덱 리스트를 요청한다
    0x54 >> sendDeckList >> 덱 리스트를 보낼때
    0x55 >> mouseCursor >> 상대가 카드를 터치하고 있을 때
0x07 >> inventory >> 

주기적으로 패킷을 보내서 접속중인지 확인한다 >> 
253 >> isConnecting >> 패킷을 쏴서 응답을 받는다.
